WHAT'S NEW IN THIS ZIP
──────────────────────

1) 📅 ROTA MODULE — Weekly staff scheduling
   • New "Rota" tab in top nav (auto-enabled once you enable the module).
   • Weekly grid: 7 days × 3 slots (Morning / Afternoon / Evening).
   • Click any + to add a shift: chef name, role, start/end time, notes.
   • Type "OFF" in the chef name to mark a day off (visually greyed out).
   • ← Prev / Today / Next → to navigate weeks.
   • Shifts store per kitchen, isolated by kitchen_id.

2) 📊 WASTE ANALYTICS — Track disposals + cost
   • New "Waste" tab in top nav.
   • KPI cards: items disposed, total quantity, estimated cost of waste.
   • Breakdown by reason (Expired / Spoiled / Damaged / Overstock / Other)
     as a horizontal bar chart with percentages.
   • Top 6 categories with waste counts.
   • Recent disposals list with dates, reasons, notes, cost.
   • Filter: Last 7 days / Last 30 days / All time.
   • ⚠️  Deleting a product now opens a "Reason" dialog instead of just
     confirming. Pick "Used up" for normal consumption (NOT logged as
     waste), or one of the waste reasons. Optionally add unit cost + notes.

3) 🔔 BROWSER NOTIFICATIONS — Expiry alerts on your device
   • Settings → Login & Alerts → "🔔 Browser Notifications" card.
   • Opt-in with "Enable notifications" button (browser will prompt).
   • Once enabled, ShelfWise sends ONE grouped notification per day
     summarising items expired + expiring in the next 3 days.
   • "Send test" button to verify it works.
   • Works on Chrome/Edge/Firefox on desktop; Safari on iOS 16.4+ once
     the PWA is installed to the home screen.

4) 🔧 PREVIOUS FIXES INCLUDED (Session 4)
   • Barcode scanner now saves & re-uses learned barcodes
   • Print Logbook is a proper in-app modal (iPhone can close!)
   • Voice modal has emergency X close button
   • Approval email sent to owner on admin approve
   • Admin panel has "📧 Test email" button

════════════════════════════════════════════════════════
  ⚠️  MANDATORY: RUN THE DATABASE MIGRATION FIRST
════════════════════════════════════════════════════════
Before you deploy, open Supabase → SQL Editor → New query,
paste the contents of `supabase/migration-7-rota-waste.sql`
and click Run.

This creates two new tables:
   • rota_shifts   — one row per (kitchen, date, slot)
   • waste_log     — one row per disposed item

Without this migration:
   • Rota screen will show "Could not load rota" toast.
   • Waste screen will show "Could not load waste data" toast.
   • Existing inventory / recipes / auth continue to work fine.
