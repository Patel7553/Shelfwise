WHAT'S FIXED
────────────

1) 🗑️  DARK MODE REMOVED
   Per your request. Theme toggle button gone from both desktop and
   mobile nav. All dark CSS overrides removed. App is always light mode
   now — nothing accidentally goes dark.
   (Code is kept as no-op stubs so we can re-add it later cleanly if
   you ever want it back.)

2) 📱 ADD/EDIT PRODUCT FORM — MOBILE LAYOUT FIX
   Expiry date and Category fields were mixing / overlapping on
   narrow phone screens. Now they stack vertically on mobile
   (grid-cols-1) and go side-by-side only on tablet+ (sm:grid-cols-2).
   Everything readable and tappable, save button always sticky.

3) 🧾 RECEIPT SCANNER — NEW EDITABLE UX
   Instead of a cramped table with tiny inputs, each detected item now
   shows as a CARD with:
     • Product name + summary (qty, unit, price, category, storage)
     • ✏️ Edit button → expands the card with full editable form:
        - Name, Quantity, Unit
        - Cost per unit, Storage
        - Category, Expiry date, Location
        - Full 14-allergen tick grid (Natasha's Law compliant)
     • ✕ button to remove that item from the import
     • Checkbox to include/skip
   Import button at the bottom shows count + total cost.

4) 🔧 RECEIPT SCANNER — BETTER ERRORS
   If the AI parse fails, the toast now shows the ACTUAL error message
   from the server (not just "Parse failed"). Stays on screen 15 sec
   so you can read it. Common cases now clearly reported:
     • EMERGENT_LLM_KEY not set → tells you
     • Image too large / bad format → tells you
     • AI returned nothing → tells you to try a clearer photo
