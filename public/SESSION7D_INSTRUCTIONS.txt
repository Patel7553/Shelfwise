WHAT'S FIXED
────────────

1) 🔧 "invalid input syntax for type integer: 2.272" ERROR
   ROOT CAUSE: Your products.quantity column is set as INTEGER.
   When the receipt scanner extracts something like "2.272 kg", the
   database rejects it because it's not a whole number.
   
   FIX: Migration 8 now automatically converts quantity from INTEGER
   → NUMERIC (which accepts decimals like 2.272 kg, 0.5 L, etc.).
   
   ⚠️ YOU MUST RE-RUN THE MIGRATION (see below). It's safe to re-run.

2) 🔧 EXPIRY DATE + CATEGORY STILL OVERLAPPING
   The 2-column layout was still kicking in on some phones. Now
   forced to single column ALWAYS — every field takes a full row,
   no more overlapping ever. Form is a bit taller but everything
   is readable and tappable.
