WHAT'S FIXED
────────────

1) 🔧 iOS DATE INPUT OVERFLOWING ITS CONTAINER
   Root cause: on iPhone Safari, <input type="date"> ignores width
   constraints because the native picker button has intrinsic sizing.
   This is why the Expiry Date box was extending outside the container.

   Fix: added strict CSS in globals.css that forces ALL date/time inputs
   to respect their container:
     • width: 100% forced
     • min-width: 0 (allows shrinking)
     • max-width: 100% (never overflow)
     • box-sizing: border-box
     • -webkit-appearance reset

   Also applied a safer default to ALL inputs so nothing overflows.

2) 🔧 SNAP DIALOG (Add Items via camera) — SINGLE COLUMN
   The Snap Label dialog still had side-by-side pairs (Qty+Unit,
   Storage+Shelf) which caused overlap on narrow phones. Now all
   fields stack vertically for consistent mobile UX.
